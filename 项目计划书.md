# 留白 Leave 完整开发流程（修订版）

## 0. 准备工作

```bash
mkdir leave && cd leave
git init
```

- 注册 Supabase 项目，记录 `SUPABASE_URL` 和 `SUPABASE_ANON_KEY`
- 注册 Vercel 账号
- 安装并认证 Codex CLI

**产品口号**：留白 Leave：把想说的话，轻轻放在这里。

**产品原则**（写入 README，作为后续所有提示词的背景约束）：
```
- 不做实时聊天，避免变成微信替代品
- 不做已读未读，降低压力
- 不做连续打卡惩罚，避免负担
- 所有内容都强调"留下来"，而不是"要求回应"
```

## 第1阶段：项目骨架（约3天）

```

配置好 ESLint 和基础目录结构（app router）。
在项目根目录创建 PRODUCT_PRINCIPLES.md，写入产品原则（不做已读未读、不做连续打卡惩罚等）。
```

验证：`npm run dev`，5个页面可跳转，底部导航正常。

## 第3阶段：Supabase Auth + profiles（约2-3天）

### 3.1 数据库 schema

```
请为"留白 Leave"项目设计 Supabase 数据库 schema：
- profiles（与 auth.users 一一对应，存昵称、头像、created_at）
- spaces（双人空间）
- space_members（关联 profiles 与 spaces）

要求：
- 不要创建 users 表，使用 Supabase 内置 auth.users
- profiles 表通过 trigger 在用户注册时自动创建一条记录
- 为每张表生成 RLS 策略
- 生成完整 SQL migration
```

在 Supabase SQL Editor 执行。

### 3.2 环境配置

```bash
npm install @supabase/supabase-js @supabase/ssr
```

`.env.local`：
```
NEXT_PUBLIC_SUPABASE_URL=你的URL
NEXT_PUBLIC_SUPABASE_ANON_KEY=你的KEY
```

### 3.3 实现认证

```
实现邮箱注册/登录功能，使用 Supabase Auth。
注册成功后自动在 profiles 表创建记录（通过数据库 trigger 实现，不在前端手写插入）。
登录后跳转到"创建/加入空间"引导页（若用户尚未加入任何空间）。
```

## 第4阶段：创建/加入双人空间（约2-3天）

```
基于已有 schema 实现双人空间功能：
- spaces 表增加 invite_code 字段（6位随机码，创建时生成）
- "创建空间"：当前用户成为 space_members 第一个成员
- "加入空间"：输入邀请码，校验空间未满（最多2人），加入成功
- 加入/创建后跳转今日页
- 在"我们"页展示邀请码、复制按钮
所有后续表的 RLS 策略需基于 space_members 关系校验 space_id 权限。
```

## 第5阶段：写一条记录 + 上传一张图片（约3-4天）

```
实现 daily_records 表与写记录功能：
字段：id, space_id, author_id, content, mood, weather, image_url（单张，非数组）, visibility, created_at

1. 创建 Supabase Storage 桶 record-images，设置 RLS 仅允许空间成员上传/读取
2. "写下今天"弹窗：文字内容 + 单张图片上传（限制 <5MB，仅图片格式，上传前预览）
3. 上传失败时给出明确错误提示，不创建记录
4. visibility 字段：'shared'(双方可见) / 'private'(仅自己/草稿)
所有操作按 space_id 隔离，使用第4阶段建立的 RLS 规则。
```

---

## 第6阶段：时间线展示记录（约2天）

```
实现记录时间线：
- 从 daily_records 读取当前空间的记录，按 created_at 倒序
- visibility='private' 的记录仅作者本人可见，对方不显示
- 卡片展示：作者头像、昵称、时间、内容、图片（若有）、天气/心情图标
- 空状态：无记录时显示温柔的占位文案，如"还没有故事呢，写下今天吧"
```

---

## 第7阶段：今日问题 - 固定题库 + 揭晓逻辑（约3-4天）

```
实现今日问题功能（固定题库方案）：
1. 创建表：question_bank（固定问题库，预先写入30条种子数据，温暖向中文问题）
2. 创建表：question_answers（id, space_id, question_index, user_id, answer, image_url, created_at）
3. 取题逻辑：question_index = 当前日期的天数序号 % 问题库总数（在代码中计算，不依赖数据库随机）
4. 用户提交答案后：
   - 若对方今日（同 question_index）未回答，仅提示"答案已记录，等待对方回答"，不展示任何人的内容
   - 若双方都已回答，双方答案同时展示
5. 答案支持附带一张图片（复用 record-images 桶逻辑）
请提供30条种子数据 SQL，语气温和、适合情侣日常。
```

```
所有数据按 space_id 隔离。
```
